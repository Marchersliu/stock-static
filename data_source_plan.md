# 数据来源规划 v2.0

## 数据分层架构

### L1 官方实时数据（Tushare Pro）
| 数据项 | API | 频率 | 可信度 |
|--------|-----|------|--------|
| 日行情（开/高/低/收/量/额） | pro.daily() | 每2分钟 | 🟢 |
| 资金流向（主力净额/超大单/大单/中单/小单） | pro.moneyflow() | 每2分钟 | 🟢 |
| 基本面（PE/PB/总市值/流通市值/换手率/量比） | pro.daily_basic() | 每2分钟 | 🟢 |
| 大盘指数（上证/深证/创业板） | pro.index_daily() | 每2分钟 | 🟢 |
| 融资融券余额 | pro.margin() | 每30分钟 | 🟢 |
| 龙虎榜 | pro.top_list() | 收盘后 | 🟢 |

### L2 权威媒体数据
| 数据项 | 来源 | 频率 | 可信度 |
|--------|------|------|--------|
| 盘前新闻/要闻 | 财联社/新浪财经/新华社 | 盘前手动更新 | 🟡 |
| 原材料价格 | 上海有色网/卓创资讯/生意社 | 每日 | 🟡 |
| 个股公告/研报 | 东方财富/同花顺 | 手动触发 | 🟡 |

### L3 AI估算/补充数据
| 数据项 | 方式 | 可信度 |
|--------|------|--------|
| 技术指标信号（MACD/KDJ/RSI） | Tushare实时计算 | 🟢 |
| 目标价距离/建仓建议 | 基于支撑位测算 | ⚪ |
| 板块轮动信号 | 资金流向+涨跌幅聚合 | ⚪ |

## 确认机制设计

### 数据更新流程
```
[定时触发] → [Tushare抓取] → [数据对比] → [有变化?] → [显示确认面板] → [用户点击"确认已阅"] → [标记为已读] → [高亮消失]
                                    ↓
                              [无变化] → [静默更新]
```

### 确认面板规则
1. **价格变化 ≥ ±2%** → 强制确认（红色高亮）
2. **主力净流入变化 ≥ ±5000万** → 强制确认
3. **止损线触发** → 强制确认 + 弹窗提醒
4. **其他变化** → 黄色提示，可选确认
5. **无变化** → 静默更新，不打扰

### 前端确认UI
- 顶部状态栏右侧显示「🔴 3条未确认」
- 点击展开确认面板，逐条展示变化内容
- 每条有「确认已阅」按钮
- 全部确认后，状态栏恢复绿色
